# 岳麓食纪 UI/UX 审计报告

文档状态：UI 审计 + 整改记录 v0.2  
最后更新：2026-04-25  
审计方法：`ui-ux-pro-max` 规则 + 当前 React/Tailwind 代码检查  
审计范围：首页地图、左侧栏、餐厅卡片、底部 BAR、榜单浮层、社区浮层、AI/饭搭子/评价弹窗

## 1. 总结

当前界面已经具备可演示的核心结构：地图主画布、左侧商家列表、详情、底部全局入口、榜单和社区浮层。首次审计发现品牌色高度依赖 `orange/yellow`，与美团黄橙心智接近，不利于“真实大学生美食地图”的差异化。

建议将品牌主色从橙黄切换为“岳麓青绿”：`#0F766E`。黄色只保留为收藏和评分的语义色。

## 1.1 整改状态

2026-04-25 已完成首轮整改：

- 已建立 `brand.*` Tailwind 语义色。
- 主链路橙色已替换为岳麓青绿。
- 评分、收藏、热度提示改为 amber 语义。
- 大面积橙黄渐变已从核心页面和组件中移除。
- 主要 icon-only 按钮补充了 `aria-label`。
- 主浮层支持 Escape 关闭和初始焦点。
- 强阴影、超大圆角和 hover scale 已明显收敛。

## 3. 主要问题

### 3.1 品牌色与美团冲突

代码中大量使用 `orange-500`、`orange-600`、`yellow-500`、`from-orange-50 to-yellow-50`。这些颜色在用户心智上接近美团、外卖、团购和促销，不适合“真实、可信、校园地图”的主品牌。

处理建议：

- 主色改为 `#0F766E`，对应 Tailwind `teal-700`。
- `orange-*` 不再作为品牌色使用。
- `yellow-*` 收敛为 `amber-*`，只用于评分、收藏、提醒。
- 地图 marker、主按钮、选中态、底部 BAR 激活态统一使用青绿色。

### 3.2 缺少品牌 token

当前颜色直接写在组件 class 中，导致后续换色会变成大量机械替换，且容易出现半套橙色、半套青绿色的混乱状态。

处理建议：

- 在 `tailwind.config.js` 或 `src/index.css` 中定义品牌语义色。
- 后续组件只使用 `brand.primary`、`brand.primarySoft`、`brand.warning` 等语义。
- 先做 token，再做全局换色。

### 3.3 圆角和阴影过重

多个主界面容器使用 `rounded-2xl`、`rounded-3xl`、`shadow-2xl`。对地图工具类产品来说，这会让界面更像营销弹窗，削弱信息密度和工具感。

处理建议：

- 餐厅卡片、按钮、输入框：`rounded-lg` 或 `rounded-xl`。
- 主浮层：最多 `rounded-2xl`。
- 减少 `shadow-2xl`，多数卡片使用 `shadow-sm` 或 `shadow-md`。
- 只有模态浮层和底部 BAR 可以保留较强阴影。

### 3.4 底部导航与右下角按钮分散

底部 BAR 现在承载榜单、语音、社区，右下角仍有饭搭子和定位。功能上可用，但视觉上形成两套全局入口，用户需要学习两个区域。

处理建议：

- 短期保留，避免扩大重构。
- 中期将“饭搭子”评估是否并入底部 BAR 或侧栏餐厅上下文入口。
- 定位按钮可继续保留右下角，因为它是地图工具动作，不是内容入口。

### 3.5 可访问性缺口

多个 icon-only 按钮依赖 `title` 或没有明确标签，例如定位、饭搭子、关闭、收藏、筛选等。键盘焦点态也主要依赖浏览器默认或 Tailwind focus ring，尚未形成统一规范。

处理建议：

- 所有 icon-only 按钮加 `aria-label`。
- 所有交互按钮保留可见 focus ring。
- 触控目标不小于 `44px`。
- 浮层打开后应考虑焦点管理和 Escape 关闭。

### 3.6 演示数据标识需要更克制

目前榜单和社区已经标注“演示功能”，方向正确。但演示卡片的视觉强调不应比真实 POI 更强，否则用户会误以为这是可信内容。

处理建议：

- 演示标签使用灰/蓝中性色。
- 真实认证、真实到店、真实评价才使用 `success` 或主品牌色。
- 社区演示评价不进入真实评分、详情评价和榜单权重。

## 4. 品牌方案

品牌主旨色：岳麓青绿 `#0F766E`。

推荐语义色：

- 主色：`#0F766E`
- 主色 hover：`#115E59`
- 主色浅底：`#CCFBF1`
- 页面浅底：`#F0FDFA`
- 主文字：`#0F172A`
- 次级文字：`#64748B`
- 边框：`#E2E8F0`
- 信息蓝：`#2563EB`
- 认证绿：`#16A34A`
- 评分/收藏 amber：`#D97706`
- 错误红：`#DC2626`

详见 [BRAND.md](./BRAND.md)。

## 5. 推荐改造顺序

1. 建立品牌 token：先在 Tailwind/CSS variables 中定义青绿品牌色。
2. 替换主链路橙色：侧栏头部、主按钮、marker、底部 BAR、榜单选中态。
3. 收敛黄色：评分和收藏保留 amber，其余黄色背景移除。
4. 降低视觉噪声：减少强渐变、强阴影、大圆角。
5. 补可访问性：aria-label、focus ring、Escape 关闭、触控尺寸。
6. 做移动端验收：375px、768px、桌面宽屏分别检查遮挡与滚动。

## 6. 验收标准

- 页面主视觉不再由橙黄主导。
- 用户第一眼能区分本产品和美团/外卖平台。
- 地图、列表、榜单、社区使用同一套品牌 token。
- 评分/收藏仍能通过 amber 清晰表达，但不承担品牌识别。
- 底部 BAR、浮层、侧栏在移动端不产生横向滚动。
- 所有 icon-only 按钮具备可读的无障碍标签。

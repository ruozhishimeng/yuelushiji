## 2026-04-22, 主链路以高德真实 POI 为准

- 背景：早期原型中商家位置和内容存在 mock，和“真实美食地图”的产品核心冲突。
- 决策：首页地图和餐厅列表以高德 `PlaceSearch` 返回的真实餐饮 POI 为主来源。
- 影响：没有真实评价、菜品、图片时展示空态，不再生成虚构内容填充。
- 后续：接入后端后，高德 POI 可作为商家基础库初始来源，再叠加平台自有评价和纠错数据。

## 2026-04-22, 建立统一 Restaurant 模型

- 背景：组件直接读取不同来源字段会导致列表、详情、marker 状态不一致。
- 决策：高德原始 POI 只能通过 `src/lib/amap/poiAdapter.js` 转成统一 `Restaurant` 模型。
- 影响：组件只消费 `Restaurant`，不直接依赖高德 POI 字段。
- 后续：后端上线后也应转换成同一模型，避免 UI 层重写。

## 2026-04-22, 地图生命周期收敛到 Hook

- 背景：地图加载、PlaceSearch、marker、选择状态放在页面里会让 `Index.jsx` 继续膨胀。
- 决策：由 `src/hooks/useAmapRestaurants.js` 负责地图生命周期和商家搜索。
- 影响：`Index.jsx` 主要负责页面编排和弹窗状态。
- 后续：如果接入 TanStack Query，可以在 Hook 内部或旁路新增 API 查询层。

## 2026-04-22, mock 数据集中且标注

- 背景：AI、饭搭子、个人中心目前还没有真实后端，但入口需要保留。
- 决策：演示数据集中在 `src/mocks/demoData.js`，相关 UI 必须标注“演示功能”或“待接入真实数据”。
- 影响：演示模块不会污染真实商家主链路。
- 后续：真实 API 接入时，优先从该 mock 模块替换，不让 mock 分散在组件内部。

## 2026-04-22, 暂不引入新的状态管理库

- 背景：当前项目仍处于原型整理期，引入全局状态库会增加迁移成本。
- 决策：继续使用 React state、props 和 hooks 管理当前前端状态。
- 影响：收藏、点赞、选中餐厅由父层或 Hook 层统一回写。
- 后续：当真实用户系统、评价系统和缓存策略稳定后，再评估是否需要更强的数据层。

## 2026-04-22, 根目录文档作为多 Agent 协作契约

- 背景：用户希望多线程甚至多 Agent 开发，需要信息流转通畅。
- 决策：新增 `AGENTS.md`、`PROJECT.md`、`ARCHITECTURE.md`、`WORKSTREAMS.md`、`ROADMAP.md`、`RUNBOOK.md`、`DECISIONS.md`。
- 影响：每个新 Agent 先读根目录文档，再认领工作流。
- 后续：每次完成重要任务后更新 `WORKSTREAMS.md` 和必要的决策记录。
